# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any repository under the [attest-framework](https://github.com/attest-framework) organization, report it responsibly via email:

**security@attest.dev**

Do **not** open a public GitHub issue for security vulnerabilities.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected repository and version (if known)
- Potential impact assessment

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 7 days
- **Fix or mitigation:** target within 30 days, depending on severity

## Scope

This policy covers all repositories in the `attest-framework` GitHub organization:

- [attest](https://github.com/attest-framework/attest) — Go engine, Python SDK, TypeScript SDK
- [attest-examples](https://github.com/attest-framework/attest-examples) — example code
- [attest-website](https://github.com/attest-framework/attest-website) — documentation site

## Supported Versions

Attest is in alpha. Security fixes are applied to the latest release only.

| Version | Supported |
|---------|-----------|
| Latest (v0.4.x) | Yes |
| Older | No |
